Black Hole Rendering

Suppose we want to render a black hole. For simplicity, let's assume the black hole isn't spinning.

For each pixel, we need to trace a light ray back in time, until we reach a light source. For now, assume the only light source is a starfield at infinity.

Then to trace this ray, we need to answer just two questions:

  • Did the light ray come from infinity or the event horizon?
  • If the former, what angle has it traversed around the black hole since it did so?

Given this data, we can pinpoint the relevant point on the starfield.

Now the symmetry of the situation comes into play. The answers to these questions depend on only two quantities:

  • The distance between the camera and the black hole.
  • The angle between the light ray and the line directly towards the black hole. (As measured by an observer whose time direction matches the Schwarzschild t coordinate.)

So we can store this information in a two-dimensional texture!

Thus the shader doesn't need to know anything about Schwarzschild geometry. It can just look up the answers it needs.

(This also means that if we want to render a charged black hole, we can simply swap out the texture.)
