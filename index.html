Curved Spacetime Raytracer

Requirements:
  • JS
  • WebGL
  • Keyboard + Mouse
Controls:
  • Click to unpause
  • WASD, Space, Shift to move
  • Mouse to look around

You've probably heard that Einstein said gravity comes from the curvature of spacetime. But what does that actually mean?

In this project, I render the curved spacetime around a planet, so you can get a visual understanding of how curved spacetime creates gravity. (Hint: Gravity happens when the time dimension gets bent toward the planet!)

A more detailed description is included in the project.

Status: Complete.

Formalization of Dragster

When speedrunning Dragster, for the Atari 2600, it is believed that the optimal time is 5.57 seconds. This is certainly an upper bound, as a time of 5.57 seconds has been achieved by dozens of players. I wish to formally prove that is is a lower bound.

Status: Incomplete.
  • Project is not yet on Github. (If you think you see it, that's an outdated version.)
  • Atari specification is complete. While it would be nice to tie the spec to something like the Visual6502 netlist, I consider this to be optional.
  • Atari specification is in the process of being connected to the Iris specification logic. Once this is done, it will be possible to start proving things about Dragster's code.
  • Documentation is nonexistent. This needs to be corrected. I'll probably use Alectryon, so I can turn the proof into a nice webpage.

4D Rendering

Just as we render a three-dimensional world to a two-dimensional screen, it should be possible to render a four-dimensional world to a three-dimensional screen.

A good example, created by someone else, is this maze game. But I want more than just line-art rendering.

Status: Failure.
  • Rendering to a voxel grid is impractical. At 1k resolution, there'd be a billion voxels.
  • Other approaches typically require polygon and/or polyhedron clipping. This seems to be too difficult to implement correctly, either for me or for library authors. I no longer trust anything that doesn't have a formal proof attached.
  • If I restrict to the 4D version of an isometric projection, the problem becomes tractable. Here's a crude prototype.
